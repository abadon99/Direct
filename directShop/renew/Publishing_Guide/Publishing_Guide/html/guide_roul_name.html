네임 규칙사항

기본 사항

id/class 네이밍 규칙

  • id의 경우는 문서에 중첩사용 금지
  • id와 class 명을 지정할 때에는 이해하기 쉽고 의미 있는 이름사용

표기형태

표기형태
종류 구분 예시
ID Selector 카멜표기법 customerService 등과 같이 대문자로 네이밍
Class Selector Hyphen 표기법 class-class 등과 같이 - 을 이용하여 네이밍
Image Underscore 표기법 img_about.gif 등과 같이 _ 을 이용하여 네이밍
파일 및 폴더 Underscore folder_file 등과 같이 _ 을 이용하여 네이밍
  • 기본적으로 Underscore (_)를 사용하여 네이밍을 표기하며, 경우에 따라서는 위의 표기형태를 적용하여 사용하도록 한다.

예약어

ID Selector

  • 아래 ID는 문서 레이아웃에 대한 예약어로 중복 사용하지 않는다.
예약어
구분 예약어 설명
사이트 레이아웃 #wrap 사이트 전체영역
#skipNav 바로가기 영역
#header 머리글 영역
#container 페이지 영역
#aside 사이드 영역
#content 본문 영역
#footer 바닥글 영역
#popXXX 팝업 페이지별 ID

Class Selector

  • 연결선택자는 3개까지를 권장한다.
  • class는 의미를 전달할수 있도록 명시적 명칭을 사용하도록 권장한다.
  • 아래 예약어는 대표적인 네비게이션, 레이아웃에 대한 가이드이며 설계가이드를 우선시 한다.
예약어
구분 예약어 설명
네비게이션 .gnb 대표메뉴 (Global Navigation Bar)
.lnb 로컬메뉴 (Local Navigation Bar)
.snb 사이드메뉴 (Side Navigation Bar)
.qnb 빠른메뉴 (Quick Navigation Bar)
.fnb 바닥글메뉴 (Footer Navigation Bar)
컨텐츠 레이아웃 .content 본문 영역
.content_header 본문 헤더 영역
.content_body 본문 내용 영역
.section 컨텐츠 기본그룹
.section_header 컨텐츠 헤더그룹
.section_body 컨텐츠 내용그룹

Img

  • 연결선택자는 3개까지를 권장한다.
  • 네이밍의 순서는 '대분류_중분류_소분류' 또는 '형태_의미_상태' 순서로 조합하여 사용한다.
  • 동일한 용도의 이미지가 여러개인 경우 마지막 단어에 숫자를 붙여서 다르게 네이밍한다.
