기본 정책 사항

기본정책

KT DIrect 리뉴얼 프로젝트

  • 기존 https://direct.kt.com/에 적용된 개발 환경에 디자인 개선 도입.
  • was 및 web 자원은 기존 다이렉트샵 자원 활용을 기준으로 한다.

반응형 구조

  • 반응형 웹으로 사이트 구성
  • window width(px)를 기준으로 하여 반응형 제작을 하며, 아래 BreakPoint를 기준으로 한다.
  • BreakPoint
    • PC 구간 : width 1024px 이상
    • Tablets 구간 : 768px to 1024px
    • Mobile 구간 : 768px 이하

was 경로

  • /ollehshop_was/src/main/webapp/WEB-INF/views/directShop/

web 경로

  • 기존 경로 :
    • css : /ollehshop_web/directShop/css/
    • js : /ollehshop_web/directShop/js/
    • images : /ollehshop_web/images/directShop/
  • 신규 제작용 경로 :
    • css : /ollehshop_web/directShop/css/ktDirect.css
    • js : /ollehshop_web/directShop/js/pub.js
      • jquery 추가 사항은 신규 파일 제작 없음
      • as-is 하단에 주석(// Direct 개선 2022) 이후에 추가
    • images : /ollehshop_web/images/directShop/renew/

퍼블리싱 규칙사항

General Formatting Rules

  • 문서의 기본구조는 html5의 규격을 준수하며, 의미에 맞게 시멘틱 마크업을 한다.
  • 들여쓰기, 빈줄삽입, 주석 등은 아래 형태로 진행한다.

들여쓰기

  • 코드 및 단락의 들여쓰기는 1탭 들여쓰기를 하며, 1탭의 크기는 4칸으로 설정한다.

빈줄

  • 문서의 가독성 및 단락 구분을 위하여 필요시 빈줄을 추가한다.
  • 단 빈줄 추가는 최대 1줄을 초과하지 아니한다.

주석

  • 문서의 주석 표시는 문서를 열람하는 모든이가 단락의 구분을 확인할 수 있도록 한다.
  • html문서의 주석은 시작주석과 끝주석을 표기하여 단락의 구분을 명확히 할 수 있도록 한다.
  • 시작주석은 <!-- 내용 -->으로 하며, 끝주석은 <!-- //내용 --> 으로 한다.
							<!-- Header -->
							<header id="header">
								<!-- Gnb -->
								<nav class="gnb">
									.....
								</nav>
								<!-- //Gnb -->
							</header>
							<!-- //Header -->

Css 규칙사항

General Formatting Rules

  • 속성값은 기본은 따옴표 '', ""를 사용한다.
  • css문서는 기본으로 들여쓰기는 사용하지 않는다. (단 미디어쿼리의 사용으로 인하여 코드구분을 하기위하여 1탭 4칸 들여쓰기는 허용한다.)
  • 속성값을 적용하기 위하여 id, class명 과 {} 사이에는 한칸 띄어쓰기를 한다.
  • css의 끝주석은 작성하지 않는다.
  • 문서의 가독성 및 단락 구분을 위하여 필요시 빈줄을 추가한다.
  • 단 빈줄 추가는 최대 1줄을 초과하지 아니한다.

세미콜론 ;

  • 속성 구분을 위한 ;은 마지막에도 사용을 한다.
							.class {margin:0; padding:0;}
